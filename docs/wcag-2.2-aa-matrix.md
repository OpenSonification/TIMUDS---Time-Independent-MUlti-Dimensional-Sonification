# WCAG 2.2 Level A and AA traceability matrix

Review date: 23 July 2026

This matrix records implementation and evidence. It does not assert WCAG conformance. Automated checks cannot evaluate every success criterion, and all manual assistive-technology entries remain pending unless an exact result is recorded.

Evidence abbreviations:

- **UT**: Vitest pure, component and axe tests.
- **E2E**: Playwright against the Vite production preview.
- **CI**: format, lint, strict type check and production build.
- **MR**: manual review or device testing still required.

## WCAG 2.2 checks requiring manual completion

The following evidence cannot be supplied by the current automated environment:

- screen-reader output and focus behaviour across the combinations in `screen-reader-test-plan.md`;
- 200% text-only enlargement and 400% browser zoom in supported desktop browsers;
- increased text spacing;
- operating-system forced colours/high contrast;
- iOS and Android reflow, touch targets and orientation;
- speech input and switch control;
- mono, one-ear and hearing-device listening;
- whether sustained tones mask speech;
- perceptual separation and listening safety;
- focus visibility and non-obscuration under browser/OS customisation.

Until these are run, rows marked “implemented” describe the code, not end-to-end conformance.
